---

## 实验一 —— 基线复现

**日期**：2026-04-08
**目标**：复现论文《Deep Hashing for Secure Multimodal Biometrics》的单模态（仅指纹）版本

### 一、实验配置

#### 数据集
| 项目 | 内容 |
|------|------|
| 数据集 | FVC2004（DB1_A, DB1_B, DB2_A, DB2_B, DB3_A, DB3_B） |
| 身份数量 | 330 人 |
| 每人图像数 | 8 张 |
| 训练 / 测试划分 | 70% / 30% |

#### 模型
| 项目 | 内容 |
|------|------|
| 骨干网络 | VGG-19（ImageNet 预训练） |
| 哈希维度 | 1024 bits |
| 二值化方法 | tanh(β·x) 连续松弛，β ∈ {1, 2, 4, 8, 16, 32} |
| 损失函数 | α·E1 + β·E2 + γ·E3（α=8, β=2, γ=2） |
| 训练策略 | 两阶段：冻结骨干网络预训练 → 端到端微调 |

#### CTM（可撤销模板模块）
| 项目 | 基线方法 | 改进方法 |
|------|----------|----------|
| 方法 | 随机位选择 | 稳定位选择（StableCTM） |
| stable_ratio | — | 0.8（从翻转率最低的 80% 位中选择） |
| 测试 G 值 | 128, 256, 512 | 512 |

#### SSTM（安全草图模板模块）
| 项目 | 内容 |
|------|------|
| 方法 | 基于 Reed-Solomon 码的模糊承诺方案 |
| 有限域 | GF(2^8)，每符号 8 bits（1 字节） |
| 码字长度 | N = G ÷ 8 个符号 |
| 安全性 | k = K × 8 bits |
| 纠错能力 | t = (N − K) ÷ 2 个符号 |
| 哈希算法 | SHA-256 |

> **关于 SSTM 实现的说明**：论文原文描述将可撤销模板 `re` 直接作为含噪 RS 码字进行解码。但由于 `re` 是任意二值数据，并非合法 RS 码字，直接解码必然失败。本实现采用等价的**模糊承诺方案**：注册时生成随机密钥 `s`，计算辅助数据 `h = re ⊕ RS.encode(s)`，存储 `SHA256(s)` 和 `h`；认证时通过对 `rp ⊕ h` 进行 RS 解码恢复 `s`。

### 三、结果分析

#### 3.1 G-S 曲线为何断崖式归零

根本瓶颈在于真实比特翻转率与 RS 码纠错模型之间的不匹配。

**数学推导**（以 G=512 为例）：

真实比特翻转率 p = 20.6%，则：

- 一个符号（8 bits）完全正确的概率：$(1 - 0.206)^8 \approx 0.158$
- 符号错误率（SER）：$1 - 0.158 = \mathbf{84.2\%}$
- 码字中期望符号错误数：$64 \times 84.2\% \approx \mathbf{54}$ 个符号

RS 码纠错能力（以 K=13, N=64 为例）：
$$t = \frac{N - K}{2} = \frac{64 - 13}{2} = 25 \text{ 个符号}$$

由于 $54 \gg 25$，RS 解码对几乎所有真实对都会失败，导致 GAR 在所有安全级别下均接近零。

#### 3.2 StableCTM 为何未能改善 G-S 曲线

StableCTM 通过从翻转率最低的位池中选择，可以小幅降低真实比特翻转率。但即使翻转率从 20.6% 降低到 15%，符号错误率仍为：

$(1 - 0.15)^8 \approx 0.272$，SER $\approx 72.8\%$，期望符号错误数 $\approx 47$ 个

仍然远超 RS 纠错能力，因此 G-S 曲线依然接近零。

**结论**：符号错误率 84% 是绝对瓶颈，CTM 层面的优化无法解决这个问题。

#### 3.3 根本原因：单模态 vs 多模态

原论文使用多模态生物特征（人脸 + 虹膜 + 指纹）。多模态融合后的哈希码更加稳定，真实比特翻转率估计低于 5%。以 p=5% 计算：

- SER $= 1 - (0.95)^8 \approx 33.7\%$
- 期望符号错误数 $\approx 22$ 个
- 对于合理的 K 值，22 个符号错误在 RS 纠错范围内

单模态指纹使用非专用的 VGG-19 在未对齐的 FVC 图像上提取特征，类内方差（Intra-class Variance）天然偏高（~20% 翻转率），导致 SSTM 的 G-S 曲线完全失效。

#### 3.4 模型质量本身良好

尽管 G-S 曲线表现差，模型本身性能优秀：
- G=512 时 EER=0.37%（未知密钥），说明判别能力强
- AUC=0.9995，ROC 曲线接近完美
- CTM 模块正确输出可撤销模板，冒充者汉明距离约 50%

**问题完全出在 SSTM 阶段**：RS 码无法处理单模态指纹哈希产生的噪声水平。

### 四、复现过程中发现的问题

| 问题 | 描述 | 修复方案 |
|------|------|----------|
| SSTM 永远返回 True | 直接对随机数据 RS 解码必然失败 → enroll 和 auth 都 fallback 到 `\x00` → 哈希相同 → 永远通过 | 重写为模糊承诺方案 |
| StableCTM 稳定池过小 | stable_ratio=0.3 → 池大小=max(512,307)=512 → 无随机性，所有用户密钥相同 | 改为 stable_ratio=0.8 → 池大小=819 |
| VGG 预训练权重未加载 | fc1/fc2 随机初始化 | 从 vgg.classifier[0] 和 [3] 复制权重 |

### 五、下一步改进方向

#### 方向 A：SSTM 引入位交错（中等难度）
在 RS 编码前对 512 bits 重新排列，使随机翻转的位尽量分散到不同符号。理论上可将符号错误数从 ~54 降至 ~13（即 102 bits ÷ 8），有望进入 RS 纠错范围。

**优点**：改动小，只需修改 SSTM
**缺点**：仍依赖翻转位均匀分布的假设，实际效果待验证

#### 方向 B：改进前端模型（高难度，治本）
将真实比特翻转率从 ~20% 降至 <5%，具体方法：
- 添加指纹对齐/配准预处理
- 使用专门的指纹特征提取网络（如细节点检测）
- 在损失函数中加入稳定性约束项

**优点**：从根本上解决问题，改进后整个系统都受益
**缺点**：工作量大，需要重新训练

#### 方向 C：改用 BCH 码（中等难度）
BCH 码直接按比特纠错，不存在符号错误率放大问题。设定 t=102 bits 纠错能力，真实对（翻转约 102 bits）理论上可以通过认证。

**优点**：从根本上避免符号错误率问题
**缺点**：需要替换整个 SSTM 的纠错码实现
