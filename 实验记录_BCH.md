# 实验记录：BCH 码改进 SSTM 安全草图模块

---

## 实验二 —— BCH 码 vs RS 码 G-S 曲线对比

**日期**：2026-04-08
**目标**：验证用 BCH 码（比特级纠错）替换 RS 码（符号级纠错）能否解决 G-S 曲线归零问题
**前置**：实验一已确认 RS 码在单模态指纹下 GAR 接近 0%（符号错误率 84%）

---

### 一、动机回顾

实验一发现 RS 码失效的根本原因：

**RS 码按符号（8 bits）纠错，随机翻转的比特会被放大为符号错误：**

$$P_{\text{符号正确}} = (1 - 0.206)^8 \approx 0.158 \implies \text{符号错误率} \approx 84.2\%$$

G=512 时，期望符号错误数 $\approx 64 \times 84.2\% = 54$ 个，远超 RS 码的纠错上限。

**BCH 码直接按比特纠错，不存在符号错误率放大问题。**

---

### 二、BCH 方案设计

#### 2.1 方案原理：部分填充模糊承诺（Partial Fuzzy Commitment）

```
注册：
  re (G=512 bits)
  → 生成随机密钥 s (k_bytes 字节)
  → BCH 编码: ecc = BCH.encode(s)
  → s_padded = s || 0...0 (共 G/8=64 字节)
  → 辅助数据: h = re XOR s_padded
  → 存储: SHA256(s) + h + ecc

认证：
  rp (G=512 bits)
  → sp_padded_noisy = rp XOR h
  → 取前 k_bytes: s_noisy
  → BCH 纠错: s' = BCH.correct(s_noisy, ecc)
  → 比对: SHA256(s') == SHA256(s)
```

#### 2.2 关键数学分析

密钥 s 只占 G bits 的前 $k$ bits，因此翻转落在 s 范围内的期望数为：

$$\text{s 内期望翻转} = 102 \times \frac{k}{G}$$

只要 $\text{s 内期望翻转} \leq t$（BCH 纠错能力），认证就能通过。

等效地，对整个 G bits 的**有效纠错能力**为：

$$t_{\text{eff}} = t \times \frac{G}{k}$$

只要 $t_{\text{eff}} \geq 102$ bits（真实翻转量），GAR 就能保持较高水平。

#### 2.3 参数对比

| 方法 | 安全性 k (bits) | ECC 开销 (bits) | 纠错能力 t (bits) | 有效纠错 $t_{\text{eff}}$ (bits) |
|------|----------------|----------------|-----------------|-------------------------------|
| RS (K=7) | 56 | 456 | 224 | 224 |
| RS (K=38) | 304 | 208 | 104 | 104 |
| RS (K=13) | 104 | 408 | 200 | 200 |
| **BCH (m=9, t=56)** | **120** | **504** | **56** | **238.9** ✅ |
| BCH (m=9, t=28) | 272 | 256 | 28 | 52.7 ❌ |
| BCH (m=9, t=15) | 376 | 136 | 15 | 20.4 ❌ |
| BCH (m=8, t=31) | 48 | 248 | 31 | 330.7 ✅ |

> **注**：RS 码的 ECC 开销占用 G bits 内部，BCH 码的 ECC 单独存储（不占 G bits）。

### 五、结果分析

#### 5.1 BCH 为何显著优于 RS

**核心原因**：BCH 码的纠错单位是比特，而 RS 码的纠错单位是符号（8 bits）。

- RS 码：102 bits 随机翻转 → 约 54 个符号出错 → 远超纠错上限 → GAR ≈ 0%
- BCH 码：102 bits 翻转落在 s（120 bits）范围内的期望仅 $102 \times \frac{120}{512} \approx 24$ bits → 远小于 t=56 → GAR = 100%

#### 5.2 拐点的理论预测与实验吻合

理论预测拐点：当 $t_{\text{eff}} = t \times G/k = 102$ 时，即：

$$k = t \times G / 102 = t \times 512 / 102 \approx 5.02 \times t$$

对于 m=9，t 从 56 递减到 34 时，k 从 120 增大到 208。实验中 GAR=50% 发生在 k=208（t=34），此时：

$$t_{\text{eff}} = 34 \times 512 / 208 \approx 83.7 \text{ bits}$$

略低于 102 bits，说明翻转分布有一定随机性，实际拐点比理论值略早，符合预期。

#### 5.3 BCH 的代价

BCH 相比 RS 的代价：
- **安全性下降**：在相同 GAR 水平下，BCH 的 k 比 RS 小（如 GAR≈50% 时，BCH k=208 bits vs RS 无法达到）
- **ECC 额外存储**：BCH 的 ecc 需要单独存储（63 bytes），而 RS 的 ecc 内嵌在 G bits 中
- **不可直接比较**：两者的安全性定义略有差异（RS 的 k 是 G bits 内的信息位，BCH 的 k 是独立密钥长度）

### 七、结论

**BCH 码比特级纠错方案成功解决了 RS 码在单模态指纹场景下的失效问题。**

1. RS 码失效根本原因：符号错误率放大（20% 比特翻转率 → 84% 符号错误率），超出所有合理 RS 参数的纠错上限
2. BCH 码通过部分填充模糊承诺方案，将有效纠错能力提升至 238.9 bits，覆盖真实翻转量 ~102 bits
3. 在 k=120 bits 安全级别下 GAR=100%，在 k=208 bits 时 GAR=50.3%，G-S 曲线形态正常
4. 该改进可作为论文的核心技术贡献，展示了针对单模态指纹高噪声特性的定制化安全草图方案
